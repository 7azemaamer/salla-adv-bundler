🔧 Salla Bundle System - Corrected Snippet

⚠️ مشكلة تم حلها:

المشكلة كانت أن متغيرات سلة مثل {{ store.domain }} لا يتم استبدالها في URL الخاص بتحميل الـ script، مما يؤدي إلى فشل تحميل الملف.

✅ الحل:

تم تعديل الكود ليقوم بتمرير متغيرات سلة عبر window.sallaStoreContext بدلاً من URL.

📋 Snippet المُصحح للاستخدام في Salla Partners Portal:

<script type="text/javascript"> var sallaStoreContext = { storeDomain: '{{ store.domain }}', storeId: '{{ store.id }}', storeUsername: '{{ store.username }}', storeEmail: '{{ store.email }}', customerId: '{{ customer.id }}', customerName: '{{ customer.name }}', customerEmail: '{{ customer.email }}', customerMobile: '{{ customer.mobile }}' };window.sallaStoreContext = sallaStoreContext; var url = 'https://app.hazemaamer.tech/api/v1/snippet/app-snippet.js'; var s = document.createElement('script'); s.type = 'text/javascript'; s.async = true; s.src = url; s.onload = function() { console.log('✅ Salla Bundle System loaded successfully'); }; s.onerror = function() { console.error('❌ Failed to load Salla Bundle System'); }; var x = document.getElementsByTagName('script')[0]; x.parentNode.insertBefore(s, x); </script>

🔄 ما تم تغييره (النسخة المتوافقة):

  1. استخدام type="text/javascript" صراحة - لتجنب مشاكل الأمان
  2. استخدام var بدلاً من const/let - للتوافق مع المتصفحات القديمة
  3. استخدام insertBefore بدلاً من appendChild - نفس طريقة التطبيقات الأخرى
  4. تجنب IIFE غير الضرورية - كود أبسط وأقل عرضة للحجب
  5. تعيين البيانات في window.sallaStoreContext - قبل تحميل الـ script
